Doclet test: Properties and Cross References

N.B.: this test has no associated application, only html instructions.

Step 1: Open javadoc provided with javafx in any browser available

To proceed please go to 

SDKROOT/docs/api
and choose file index.html

Step 2: "properties" are listed in separate section

select package javafx.animation in upper-left frame

make sure class Animation present in the list of package members (bottom-left frame)

select class Animation in the list of package members

in right frame, check sections Property Summary,Field Summary,Constructor Summary, Method Summary are exist

Step 3: "summary" and "detail" sections

check rate property present in both Property Summary and Property Detail sections

Step 4: inherited documentation

check setRate and getRate methods has same non-empty documentation text

Step 5: cross-reference for property

in "Property Summary" section, click on autoReverse property

make sure autoReverse property decription shown in "Property Detail" section

Step 6: cross-reference for field

in "Field Summary" section, click on INDEFINITE field

make sure INDEFINITE field decription shown in "Field Detail" section

Step 7: cross-reference for constructor

in "Constructor Summary" section, click on Animation() constructor

make sure Animation() constructor decription shown in "Constructor Detail" section

Step 8: cross-reference for method

in "Method Summary" section, click on autoReverseProperty() method

make sure autoReverseProperty() method decription shown in "Method Detail" section
