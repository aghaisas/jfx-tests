Doclet test: Visibility Rules

N.B.: this test has no associated application, only html instructions.

Step 1: Open javadoc provided with javafx in any browser available

To proceed please go to 

SDKROOT/docs/api
and choose file index.html

Step 2: Validate public classes and interfaces are listed

select package javafx.animation in upper-left frame

make sure class Animation present in the list of package members (bottom-left frame)

select class Animation in the list of package members

make sure class description of some kind appear in right frame

make sure interface Interpolatable present in the list of package members (bottom-left frame)

select interface Interpolatable in the list of package members

make sure interface description of some kind appear in right frame

Step 3: public and protected members are treated as visible

select package javafx.animation in upper-left frame

select class Animation in the list of package members

make sure field INDEFINITE present in javadoc in right frame

make sure method setCycleDuration present in javadoc in right frame

Step 4: package local entities are not listed

select package javafx.animation in upper-left frame

select class Transition in the list of package members

make sure member parent is not appear in javadoc in right frame

Step 5: members marked as "implementation internal" are not listed

select package javafx.animation in upper-left frame

select class Transition in the list of package members

make sure members with names beginning with impl_ does not appear in javadoc in right frame

Step 6: properties with any visible accessors should be listed regardless of their own visibility

select package javafx.animation in upper-left frame

select class Animation in the list of package members

make sure property onFinished present in javadoc in right frame
