Multitouch Shape Test: Ellipse, Zoom / Rotate gesture, not null fill

Prerequisites: Installed FX runtime, multitouch hardware support

  • Uninstalled all previous version of FX sdk & runtime;
  • Installed FX runtime
  • Please see SceneGraphTests/README-mutitouch.html file to get basic knowledges about multitouch.

Test workflow:

  1. Take a look at the test application (MultiTouchApp)
  2. If test runs on the computer with touch screen push toggle button "Direct" (for touch pad using this button must be unselected)
  3. Use Node.addEventHandler to receive multitouch events:
    1. Select "ADD_HANDLER" (it is default value) in the choice box
    2. Push button "AddEllipse"
    3. Check that a random half-transparent ellipse is painted on the scene
    4. Do Zoom / Rotate gesture for the shape. Be sure that center between touches is inside the shape
    5. In case shape stays half-transparent after gesture start - test fail
    6. In case shape is not scaled during gesturing - test fail
    7. In case shape is not rotated during gesturing - test fail
    8. In case shape is not half-transparent after gesture end (no touches) - test fail
    9. Push button "ClearScene"
  4. Use Node.addEventFilter to receive multitouch events:
    1. Select "ADD_FILTER" in the choice box
    2. Push button "AddEllipse"
    3. Check that a random ellipse is painted on the scene
    4. Do zoom gesture for the shape
    5. In case shape stays half-transparent after gesture start - test fail
    6. In case shape is not scaled during gesture - test fail
    7. In case shape is not rotated during gesturing - test fail
    8. In case shape is not half-transparent after gesture end (no touches) - test fail
    9. Push button "ClearScene"
  5. Use Node.setOn<EventType> handler to receive multitouch events:
    1. Select "SET_ON_EVENT_HANDLER" in the choice box
    2. Push button "AddEllipse"
    3. Check that a random ellipse is painted on the scene
    4. Do zoom gesture for the shape
    5. In case shape stays half-transparent after gesture start - test fail
    6. In case shape is not scaled during gesture - test fail
    7. In case shape is not rotated during gesture - test fail
    8. In case shape is not half-transparent after gesture end (no touches) - test fail
  6. In case of successful verification press pass, otherwise fail
