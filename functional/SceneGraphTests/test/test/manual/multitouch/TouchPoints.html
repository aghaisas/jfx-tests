Touch Points Test

Prerequisites: Installed FX runtime, multitouch hardware support

  • Uninstalled all previous version of FX sdk & runtime;
  • Installed FX runtime
  • Please see SceneGraphTests/README-mutitouch.html file to get basic knowledges about multitouch.

Test workflow:

  1. This test is applicable only for touch screen. In other case press "Passed"
  2. Take a look at the test application. It looks like (without colored circles after start):
    • Red circle marks pressed TouchPoint (stay at touched point and fade);
    • Green circle marks moved TouchPoint (follow a touch point);
    • Blue circle marks stationary TouchPoint;
    • Grey circle marks released TouchPoint (stay at released point and fade);
  3. When you use two-fingers for touch the line must be drawn between touch points. Also the center between touch points is marked
  4. Use touch screen and check the position & color of the circles. Sometimes hardware touch screen can send strange events. This usually associated with the dirty screen or features touch screen with your finger
  5. So check that general touch screen events are correct. And you can see circles with rights colors at right places (where you touch, move, hold & release finger)
  6. Push toggle button "TouchPress/Release"
  7. Touch screen on the scene and move finger to left outside the scene. Then release finger from touch screen
  8. See at the message log. Check the X coordinate of the point in TOUCH_RELEASED event. In case X coordinate >= 0 test fail. 
    TouchEvent [..., eventType = TOUCH_RELEASED, ... , touchPoint = TouchPoint [state = RELEASED, id = 1, target = VBox@4ba046fe, x = -163.0, y = 318.0]
  9. Push button "ClearLog"
  10. Touch screen on the scene and move finger to up outside the scene. Then release finger from touch screen
  11. See at the message log. Check the Y coordinate of the point in TOUCH_RELEASED event. In case Y coordinate >= 0 test fail. 
    TouchEvent [..., eventType = TOUCH_RELEASED, ... , touchPoint = TouchPoint [state = RELEASED, id = 1, target = VBox@4ba046fe, x = 136.0, y = -198.0]
  12. In case of successful verifications test pass, otherwise fail
