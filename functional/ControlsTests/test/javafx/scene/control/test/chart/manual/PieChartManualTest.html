PieChart animation tests

All charts could be animated, or not (it is switched with a "animated" property). The aim of this test is to check animation of chart, and common rendering. When it is animated, actions like adding, removing and changing data values, will be done with animation (it takes time about 700 ms), or without it (immediately).

Test case 1 : animated

  • If it is not true, switch "animated" property to "true"
  • Remove all the data (use corresponding button) - data will be removed from index 0.
  • Add some data several times (use corresponding button) - data will be added at index 0.
  • Change data items (use corresponding button) - value of data item at index 0 should change (randomly : increase or decrease).

Test case 2 : not animated

  • If it is not false, switch "animated" property to "false"
  • Remove all the data (use according button).
  • Add some data several times (use according button).
  • Change data items (use according button).

Test case 3 : common rendering

  • Try to rotate chart, changing "start angle property"
  • Try to change rotation direction, changing "clockwise" property
  • Try to change chart sizes, changing "prefWidth" and "prefHeight" properties
  • Try to change header, changing a "title" property
  • Try to change sides of title and legend, changing "titleSide" and "legendSide" properties
  • Try to hide and show legend, changing "legendVisible" property

Please, note

Some cases, when part of charts (legend, labels) can disappear - that is normal behavior.
  • Legend could disappear, when there is no enough space
  • Some labels could disappear, when there is not enough of space for their allocation
  • All labels could disappear, when there is not enough space
Comments on testing :
  • About animation : it is expected, that animation behaves itself smoothly. It is expected, that there are no jumps, there are no rendering artifacts during animation, and it behaves itself correctly.
  • Please, verify rendering at each step: no overlapping content, no graphical artifacts, etc.

Help

  • To add, change, or remove data, use according buttons - changing will be done at index 0.
  • To change a property value, you can use according control (TextField for String properties, ToggleButton for boolean properties, ChoiceBox for enums, Slider for double properties). To change a value, set it using some of mentioned control and switch on binding (press toggle button "unbinded" to change its state to "binded").
  • You can use Reset button to start again. The whole scene will be dropped.
