Directory chooser manual test

Description

JavaFX provide common access to native file and directory choosers (they are different on different Operation Systems, and each can have its specific). File choosers allow to select file for saving and opening. We will test it.

Note

  • There is a "No errors" label in the bottom of the scene. If any error happens, in the place of this label will be shown an indication about error, and error will be printed into error output, and shown in the tooltip of the label. Please, check this label, on each step, and fail the test, if you see any exception.
  • Each of buttons ["Save file FX (modal dialog)", "Save file with extension FX (modal dialog)", "Open File FX", "Open File with extension FX", "Open Multiple Files FX", "Open Multiple Files With extensions FX"] (6 buttons) open a dialog for file opening or closing. I will call them "FILE DIALOG BUTTONS"
  • If a button has text "(modal dialog)", then a dialog will be opened as modal window, and will block the window with buttons.
  • If a button has notion "with extensions", it means, that extension "*.extension" will be added for saved or opened files.
  • Save dialogs will save files to open them later with open dialogs, so navigate to a directory, which you have rights to write in. Remove them after testing is finished.

Step 1: Simple open/close case.

  • For each FILE DIALOG BUTTON do actions below :
  • Just press the button to open the dialog.
  • Check, that modal dialogs block the underlying window, and non modal don't block.
  • Close appeared dialog.
  • Check, that returned result is null.
  • Check that there are no exceptions, everything works fine

Step 2: Title could be set.

  • In a text field for Title type "Custom title". It will set a title for file dialog.
  • For each FILE DIALOG BUTTON do actions below :
  • Open a dialog, check that title of file dialog is "Custom title", close the dialog.
  • Check that there are no exceptions, everything works fine

Step 3: Saving files.

  • Click button "Save file FX (modal dialog)".
  • Type name "file1.extension", and save.
  • Check, that file "file1.extension" was physically created (via file browser, of via terminal).
  • Check, that result of the dialog points to correct location of file "file1.extension".
  • Click button "Save file with extensions FX (modal dialog)".
  • Navigate, to the place, where previous file was saved.
  • Mac only : press on drop down button, to see the folder content (by default you see minimized version of dialog).
  • If you are NOT ON Mac : shown files should be filtered via "*.extension" extension, so you should see "file1.extension" and folders.
  • If you are ON Mac : you should see "file1.extension", other files and folders.
  • Type file name "file2", and press save.
  • WIN : Check, that result of the dialog points to correct location of file "file2.extension".
    • Linux

      Check, that result of the dialog points to correct location of file "file2" without extension

    • Mac

      Check, that result of the dialog points to correct location of file "file2.extension"

  • WIN : Check, that file "file2.extension" was physically created in that location.
    • Linux

      Check, that file "file2" (without extension) was physically created in that location

    • Mac

      Check, that file "file2.extension" was physically created in that location.

  • Open the same dialog again, and navigate to the previous location.
  • Type file name "file1", and press save.
  • WINDOWS only : check, that an alert (warning) about overwriting an existing file is shown.
  • Check, that result of the dialog points to correct location of file "file1.extension".
  • Check that there are no exceptions, everything works fine.

Step 4: Saving files with predefined name.

  • Print "file3" in a text field for Initial file name.
  • Click button "Save file with extensions FX (modal dialog)".
  • Check, that name of the file for saving is set to "file3" or "file3.extension".
  • Press save.
  • Check, that result of the dialog points to correct location of file "file3.extension", and that file exists physically.
  • Check that there are no exceptions, everything works fine.

Step 5: "New Folder" option in a save menu.

  • Click button "Save file FX (modal dialog)".
  • Mac only : click on drop down button to see the extended form of dialog (by default you see a minimized version of dialog).
  • Check, that "New folder" button is present in a dialog. (in the top left on windows, in the bottom left on the mac), or "Create folder" in linux.
    Note that on Windows system you have to select the logical drive first. E.g. double click with mouse the 'C:' label and after that the 'New folder' button should appear.
  • Close dialog
  • Click button "Save file with extensions FX (modal dialog)".
  • Check, the same (button about new folder creating exists).

Opening files.

Step 6. Opening file.

  • Click button "Open file FX".
  • Navigate to place, where "file*.extension" were saved.
  • Open any of files - file1 or file2, or file3 from previous tests (select it via mouse click on the file).
  • Press open.
  • Check, that result of the dialog points to correct location of that file and extension "*.extension" is added.
  • Check that there are no exceptions, everything works fine.

Step 7. Opening file with extension.

  • Click button "Open file with extensions FX".
  • Navigate to place, where "file*.extension" were saved.
  • Windows, Linux : Type file name "file1" | MacOS : select file1.extension.
  • Press open.
  • Check, that result of the dialog points to correct location of file "file1.extension" and correct extension is added to the end.
  • Click button "Open file with extensions FX".
  • Navigate to place, where "file*.extension" were saved.
  • Check, that files are filtered : only "file*.extension", and some folders (possibly, they are not filtered out) are shown.
  • Select any file from list "file*.extension", and open it
  • Check, that result of the dialog points to correct location of file, which you've opened and correct extension is added to the end.
  • Check that there are no exceptions, everything works fine.

Step 8. Opening file.

  • Click button "Open Multiple files FX".
  • Navigate to place, where "file*.extension" were saved.
  • Open files: - file1 and file2, from previous tests (select it via mouse click on the file).
  • Press open.
  • Check, that result of the dialog points to correct location of those files, separated with ";" and extension "*.extension" is added to the each one.
  • Check that there are no exceptions, everything works fine.

Step 9. Opening file with extension.

  • Click button "Open Multiple Files With extensions FX".
  • Navigate to place, where "file*.extension" were saved.
  • Check, that only files "file*.extension" are shown (and folders)
  • Select file1 and file2, and press open.
  • Check, that result of the dialog points to correct location of file "file1.extension" and "file2.extension" and correct extension is added to the end.
  • Check that there are no exceptions, everything works fine.
