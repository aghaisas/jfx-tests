Stage parameters test

Description

Stage - is a window, which is provided by operation system for an application. It can be in different states : maximised, iconified, fullscreen, and other, and also has paramteres : x, y, width, height. Some parameters you can change, and some of them you can only observe for changes. We will test changing the changable parameters and check, that we are notified about changable parameters.

Note

You can see text fields, which contain actual value of the property, which is mentioned on the left side from the text field. You can see previous values, in tooltip, related to each textfield. During different manupilations with window, check please, that scene doesn't blink, doesn't become black, doesn't disappear.

Step 1: Fullscreen.

  • Track "fullscreen" property value, while doing the next manipulation
  • Click on toggle button "Fullscreen state : false",
  • Verify, that scene is in fullscreen mode, and property is set to true.
  • Unselect the toggle button, and verify the property value is set to false.

Step 2: Position.

  • Track "x" and "y" property values, while doing the next manipulation
  • Change scene location, and check that x and y are changing correspondingly.
  • If you have several screens, move from one screen to another.
  • Note : x and y could be negative, depending on which of few monitors is the leading one.

Step 3: Resizing.

  • Track "width" and "height" properties, while doing the next manipulation
  • Resize scene , and check that width and height are changing correspondingly.

Step 4: Iconification.

Iconified - it is when application is collapsed on a task bar (in windows or linux), or to the dock in mac os. ]
  • Check, that "iconified" property, is false.
  • Collapse scene ("_" button in the top right corner of stage in windows, "-" round button on mac), and expand it back.
  • Check, that value is set to false, still.
  • Look at the history of values, and check, that it was set to true. (to look at history - move mouse on a text field, to see a tooltip; you cannot see directly, when it is true, because it is true, when scene is iconified).
  • Only for Mac: (mac supports applications collapse on CMD+H keys combinations). Press CMD+H, check that application would hide. Look at history of values and check that no additional 'true/false' pair appreared.

Step 5: Maximization.

Maximized - it is when application stage is as huge as possible (use "+" on mac, or rectangular button on windows in the top right corner of the stage)
  • Check taht "maximized" property is false.
  • Maximize stage, and check, that value of property is changed to true.
  • Shrink stage, and check, that value is set to false back.

Step 6: Focused.

  • During this step check, that "focused" property is true, when stage is in focus.
  • Focus a stage of some other application, and check, that value is set to false.
  • Focus stage with testing scene again, and check, that value is set to true.

How to change properties' values.

You can see buttons "unbinded". They allow to bind/unbind values of slider (valueProperty - double values), textField (textProperty - string values), toggleButton (selectedProperty - boolean properties), with double, string, boolean properties of tested subject (now - a stage). BIDI - corresponds to a bidirectional binding, UNIDI - corresponds to a unidirectional binding. We will use a bidirectional binding (Toggle buttons after BIDI labels). Select all 3 the most right "unbinded" toggle buttons, so that they change value to "binded".

Step 7: Title.

  • Check "title" property's value during this step, and actual title of the stage,
  • Change title to "Custom title" (type text into text field, when it is binded).
  • Check, that property changed, and stage title is changed.

Step 8: Resizable.

  • Check, that "resizable" property is true by default.
  • Change value to false, using according toggle button between "BIDI" and "binded" texts.
  • Try to resize stage. You cannot be able to do that.
  • Change value back to true.
  • Try to resize stage. You must able to do that, and check, that width and height are also changing.

Step 9: Opacity.

  • Looking on the text field related to "opacity" property, check, that it is equal to 1.0.
  • When property of slider is "binded" move slider, and check, that opacity is changing in range [0..1].
  • Check, that stage is changing opacity, and value of property is changing.
  • Note, that when you move slider to values more than 1.0, or less than 0.0, exception "java.lang.IllegalArgumentException: Alpha should be in the range [0f..1f]" is thrown. - it is normal.
